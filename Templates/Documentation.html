Documentation

Cette API permet de scraper des données sur Alibaba, de les nettoyer et de les stocker dans une base PostgreSQL. Elle est containerisée avec Docker et surveillée avec Prometheus et Grafana.

Endpoints de l'API

Cette section présente l'ensemble des endpoints disponibles dans l'API. Chaque endpoint est détaillé avec sa méthode HTTP, sa description, ses paramètres et la réponse attendue. Ces endpoints permettent d'interagir avec l'application, d'effectuer des opérations de scraping, de récupérer des données et d'afficher les différentes pages du site.

Page d'accueil

GET /

  • Description : Affiche la page d'accueil de l'application.
  • Réponse : Renvoie le template Home.html.

Pages HTML du site

GET /tb

  • Description : Affiche un tableau de bord Grafana intégré.
  • Réponse : Renvoie une page HTML contenant un iframe Grafana.

GET /choix

  • Description : Affiche la page de sélection des options de scraping.
  • Réponse : Renvoie le template Choix.html.

GET /Categorie

  • Description : Affiche la page de scraping des catégories Alibaba.
  • Réponse : Renvoie le template Scrap_Cat.html.

Pages HTML du site

GET /Produit

  • Description : Affiche la page de scraping des produits Alibaba.
  • Paramètres : db: Session (dépendance)
  • Réponse : Renvoie le template Scrap_Produit.html avec les catégories disponibles.

GET /Documentation

  • Description : Affiche la documentation de l'API.
  • Réponse : Renvoie le template Documentation.html

GET /APropos

  • Description : Affiche la page "À propos".
  • Réponse : Renvoie le template A-propos.html.

Endpoints API pour la gestion des catégories

GET /categories/

  • Description : Récupère la liste de toutes les catégories.
  • Réponse : Renvoie un JSON contenant toutes les catégories stockées en base de données.

POST /scrape

  • Description : Lance le scraping des catégories Alibaba et stocke les résultats en base de données.
  • Paramètres : request: Request, db: Session (dépendance)
  • Réponse : Met à jour la base de données avec les nouvelles catégories et renvoie Scrap_Cat.html.

Endpoints API pour la gestion des produits

GET /produits/{title}

  • Description : Récupère les produits associés à une catégorie donnée et les stocke en base de données.
  • Paramètres : title: str (Nom de la catégorie), request: Request, db: Session (dépendance)
  • Réponse : Scrape les produits de la catégorie et met à jour la base. Renvoie Scrap_Produit.html avec les produits récupérés.

GET /InfoProd

  • Description : Affiche une page pour rechercher les informations détaillées sur un produit.
    Réponse : Renvoie le template index.html

POST /InfoProd/URL

  • Description : Scrape les informations détaillées d'un produit via son URL Alibaba.
  • Paramètres : url: str (URL du produit à scraper)
  • Réponse : Renvoie un JSON contenant les détails du produit.

Endpoints ​de recherche

GET /search

  • Description : Affiche une page de recherche de produits Alibaba.
  • Réponse : Renvoie le template Recherch_prod.html.

POST /search/Prod

  • Description : Effectue une recherche de produits Alibaba en fonction d'un mot-clé.
  • Paramètres : keyword: str (Mot-clé recherché), max_pages: int (par défaut 2, nombre de pages de résultats à scraper)
  • Réponse : Renvoie le template Recherch_prod.html avec les résultats de la recherche.

Endpoints de récupération des données stockées

GET /categories/

  • Description : Récupère la liste des catégories enregistrées en base.
  • Réponse : Renvoie un JSON contenant la liste des catégories avec titre et lien.

GET /get_products

  • Description : Récupère la liste des produits enregistrés en base.
  • Réponse : Renvoie un JSON contenant la liste des produits avec titre, lien, prix, et quantité minimale de commande.

Installation et Exécution

Prérequis

  1. Python 3.10+
  2. Docker et Docker Compose
  3. Git

 

Acquisition du projet

    Elle se fait via git en clonant le repository qui contient le projet avec :
  • git clone https://github.com/GanItachi/Alibaba_Scrapping.git

Exécution

    Pour ce faire il faut lancer le docker, c'est à dire démarrer tous les services avec Docker Compose​ avec la commande suivante:
  • docker-compose up --build

N.B: Veillez à être à la racine du projet avant de lancer le docker (cd Alibaba_Scrapping)
